# review-prompt.md 修正計画

## Context

外部レビューで指摘された問題点を修正し、レビュープロンプトの精度と実用性を向上させる。

## 修正内容

### 1. 「前回レビューでの指摘事項（全て対応済み）」→「対応方針」に変更

**理由**: 「全て対応済み」と断定すると、レビューが検証ではなく追認になる。

**修正内容**:
- セクション名を「前回レビューでの指摘事項（全て対応済み）」から「対応方針（要確認）」に変更
- 未反映箇所を明示:
  - `具現化＝固定化` → plot-final.md:40にあるがplot-context.md:46には未反映
  - `外部干渉と自己切断の差` → plot-final.md:84にあるがplot-context.md:155には未反映

### 2. 出力形式に「根拠の明示」を必須化

**修正内容**:
- 出力形式に「各指摘には必ずファイル名と行番号を付ける」ことを明記
- 指摘例を追加

### 3. 3ファイル横断整合性チェックを追加

**修正内容**:
- レビュー観点に「3ファイル間の整合性」を追加
- 具体的なチェック項目:
  - 同じ設定が3ファイル間で同じ粒度で反映されているか
  - 伏線の配置が3ファイル間で整合しているか

### 4. 分量設計の記述を正確化

**修正内容**:
- 「長編（30,000字以上）」の記述に、writing-guide.md:25の「28,500〜34,000字」という正確な数値を併記

### 5. 出力形式の順序を変更

**修正内容**:
- 現在: 総評 → 修正が必要な点 → 改善推奨点 → 良い点 → 最終判定
- 変更後: **修正が必要な点** → 改善推奨点 → 総評 → 良い点 → 最終判定
- 重大な不整合が埋もれないよう、修正点を先頭に配置

## 修正後のレビュー観点（追加部分）

```markdown
### 7. 3ファイル間の整合性

- [ ] 同じ設定が3ファイル間で同じ粒度で反映されているか
  - 例：plot-final.mdの「具現化＝固定化」がplot-context.md、writing-guide.mdにも反映されているか
- [ ] 伏線の配置が3ファイル間で整合しているか
  - 例：7年後エラの孤独の伏線が全ファイルで一貫しているか
- [ ] 台詞や設定の重要な表現が3ファイル間で一致しているか
```

## 修正後の出力形式

```markdown
## 修正が必要な点
1. [ファイル名:行番号] 問題点と修正案
2. ...

## 改善推奨点
1. [ファイル名:行番号] 改善案
2. ...

## 総評
（全体的な評価）

## 良い点
1. （評価できる点）
2. ...

## 最終判定
（商業レベルかどうかの判定）
```

## 検証方法

1. 修正後のプロンプトをLLMに入力
2. 3ファイルをレビューさせ、指摘にファイル名/行番号が付いているか確認
3. 横断チェックが実行されているか確認
